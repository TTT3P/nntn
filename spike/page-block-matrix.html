takeaway 01 · invest hard

activity-log + metric-kpi คือ workhorses

activity-log ใช้ทุกหน้า (14/15 · ยกเว้น count-sheet print) · metric-kpi ใช้ 12/15 · พวกนี้ต้อง แข็งแรงสุด · loading/empty/error states · responsive · accessibility · ทดสอบกับ data ทุกชนิด

takeaway 02 · niche but high value

progress-tier + low-stock + timeline

ใช้ใน 7-8 หน้า · ส่วนใหญ่ secondary · แต่หน้าที่ใช้เป็น primary (kanban, stock-dispense, ops-console) reliance สูง · build เป็น atomic primitives ที่ compose ได้ ไม่ใช่ one-off

takeaway 03 · skip for now

heatmap + chart-line · 2 หน้าเท่านั้น

heatmap-grid และ chart-line ใช้น้อย (2 หน้า each · sales + platform-health) · ดี enough version พอ · ถ้าเวลาไม่พอ ทำเป็น static SVG ก่อนได้ · refactor ทีหลังเมื่อเห็น pattern
