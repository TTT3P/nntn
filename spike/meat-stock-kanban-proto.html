rpc_meat_transfer · drag-to-move contract

Drag-drop UI → 1 rpc call · atomic · ออก 2 movement events (OUT จาก source, IN to dest) · ไม่ใช่ bulk update columns · ใช้ pattern เดียวกับ rpc_po_receive เพื่อให้ข้อมูลเดินทางผ่าน movement log เป็น audit trail เสมอ

client · drag drop react-dnd

// onDrop handler async function handleDrop(bag, toWarehouseId) { // optimistic UI — move card locally moveBagInState(bag.id, toWarehouseId); try { await rpc("rpc_meat_transfer", { p_actor: currentUser, p_bag_id: bag.id, p_to_wh: toWarehouseId, p_reason: "manual transfer" // optional }); } catch (err) { // rollback — move card back rollbackBag(bag.id); toast.error(err.message); } }

VALUES (..., 'transfer_out'); -- emit IN to new warehouse INSERT INTO stock_movements(...) VALUES (..., 'transfer_in'); -- update bag.warehouse UPDATE meat_bags SET warehouse_id = p_to_wh WHERE id = p_bag_id; RETURN jsonb_build_object('ok', true); END; $$ LANGUAGE plpgsql; -- trigger updates sm (source -1, dest +1 per bag)
