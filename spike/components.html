ทำไมเน้น mobile ก่อน

  • น้องแพ็ค/แปรรูปใช้มือถือขณะชั่งเนื้อ มือเปียก — ปุ่มต้อง ≥ 44px touch
  • ของเหลือกับผลชั่งกรอกที่เกิดขึ้นระหว่างแพ็ค ไม่ใช่หลังจบงาน
  • Desktop ใช้เฉพาะใน Admin / Archive / ตรวจนับสิ้นเดือน
  • หน้า hub-delivery (Phase 1D) จะเป็น mobile-first + progressive disclosure ให้ B2/B6 bug หายไปเอง
