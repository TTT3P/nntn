Design system + ต้นแบบ 8 หน้าจอ สำหรับร้านเนื้อตุ๋น NNTN — ตีกรอบ visual language แบบ heritage (เขียวเข้ม + ทองเพชร) และวางแนวทางแก้ปัญหาสต๊อก 3 ระบบ (แปรรูป / non-meat / catch-weight) ให้อยู่ภายใต้ UI เดียวกัน

01

Show gaps, don't hide them

ใช้ ∅ (NULL) เป็น first-class state — ช่องว่างไม่ใช่ศูนย์, มันคือข้อมูลที่ยังไม่ได้กรอก และต้องเห็นชัด

02

Partial dispense = first-class

เบิก 12 kg ได้จริง 9 kg? แสดง 9/12 พร้อม badge "partial" — ไม่บังคับให้แก้ตัวเลข

03

Every write is traceable

ทุก action ลง activity log อัตโนมัติ — เห็นได้ในหน้า Dashboard และ Stock Detail
