ขั้นตอน

วิธีทำงานแต่ละ channel

1. Grab Delivery

  • 1 Login merchant.grab.com
  • 2 ไปที่ Reports → เลือกเดือนที่ต้องการ
  • 3 Download 3 ไฟล์: Menu Item Summary, Modifier Summary, Daily Summary
  • 4 วางในโฟลเดอร์ nntn-sales-analytics/APR2026_receipts/
  • 5 รัน python3 nntn_db_build.py → ข้อมูลเข้า DB อัตโนมัติ ⚠️ CSV มีแถว "รวม" ท้ายไฟล์ — script กรองออกให้แล้ว ไม่ต้องลบมือ

2. FoodStory / LINE MAN

  • 1 เปิด Chrome → ไป owner.foodstory.co/th/summary → Login
  • 2 กด F12 → แท็บ Console
  • 3 พิมพ์คำสั่ง Set วันที่ก่อน (ดึง CSRF token + POST /api/setDate) ⚠️ ต้องส่ง X-CSRF-TOKEN ทุกครั้ง ถ้าขาดจะได้ Error 500
  • 4 Copy URL จาก Network tab (แท็บ Network → ค้นหา "salebybilldetail") → ดึงข้อมูล ⚠️ URL ต้องมี 29 column params ครบ ห้ามพิมพ์ใหม่เอง ต้อง copy จาก Network tab เท่านั้น
  • 5 ข้อมูล ~1,000 rows แบ่งเป็น 5 ชุด → Claude อ่านแล้ว import เข้า fs_items

3. บันทึกการใช้เนื้อ (ครัว)

  • 1 ทีมครัวชั่งเนื้อแต่ละประเภทตอนสิ้นวัน
  • 2 ไทน์ส่งข้อมูลมาในรูปแบบ text ให้ Claude
  • 3 Claude กรอกลง daily_meat_use_report.xlsx อัตโนมัติ (ถูก sheet + row)
  • 4 ชายโครง: บันทึกแยก โกเบ vs ตุ๋น → คำนวณ yield% ให้อัตโนมัติ 📌 Yield เฉลี่ย: โกเบ 48% / ชายโครงตุ๋น 52% ของน้ำหนักดิบ
