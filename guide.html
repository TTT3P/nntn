1 เข้าสู่ระบบ
ttt3p.github.io/nntn/login.html
🥬 NNTN สต๊อกครัว
เปิดหน้า Login →
👤 ชื่อผู้ใช้: ชื่อจริงภาษาอังกฤษ 🔑 รหัสผ่าน: เบอร์โทรของคุณ
ขั้นตอนการ Login
  1. พิมพ์ชื่อผู้ใช้

    ช่อง "ชื่อผู้ใช้" — พิมพ์ชื่อภาษาอังกฤษของคุณ เช่น aim tor mile (ตัวใหญ่เล็กไม่สำคัญ)

  2. พิมพ์รหัสผ่าน

    ช่อง "รหัสผ่าน" — ใส่เบอร์โทรของคุณ กด Enter หรือกดปุ่ม "เข้าสู่ระบบ"

  3. ระบบพาไปหน้าหลักอัตโนมัติ

    ถ้า login สำเร็จจะเข้าสู่ระบบทันที — มุมบนจะแสดงชื่อของคุณ

⚠️
ถ้าขึ้น "ชื่อผู้ใช้หรือรหัสผ่านไม่ถูกต้อง" ให้แจ้งไทน์ทันที — ห้าม login ด้วยบัญชีคนอื่น
2 รับเนื้อสด
📍
ใช้เมื่อ: รับวัตถุดิบเนื้อเข้าคลัง A — กรอกทุกครั้งที่มีเนื้อเข้าใหม่
🛒 รับเนื้อสด 🔥 หม้อตุ๋น ♻️ แปรรูป 📊 สต๊อกปัจจุบัน
ขั้นตอน
  1. เปิดหน้า Meat Stock → tab 🛒 รับเนื้อสด

    meat-stock/index.html

  2. กรอกข้อมูลแต่ละช่อง
    วันที่รับเข้าระบบใส่วันนี้ให้อัตโนมัติ — เปลี่ยนได้ถ้าบันทึกย้อนหลัง
    เลขที่ใบซื้อ/ใบส่งเช่น PO-001 — กรอกตามเอกสารที่ได้รับ
    ประเภทเนื้อเลือกจาก dropdown — มีเฉพาะเนื้อที่ใช้งานอยู่
    วัน Lotระบบใส่วันนี้ให้ — ตรงกับวันผลิต/รับ
    จำนวนถุงตั้งเป็น 1 — เปลี่ยนเป็นจำนวนถุงที่รับจริง
    น้ำหนักรายถุงกรอกน้ำหนัก (กรัม) ทีละถุง เช่น 5200 = 5.2 กก.
  3. ถ้ามีเนื้อหลายชนิดในครั้งเดียว

    กดปุ่ม + เพิ่มรายการเนื้อ เพื่อเพิ่ม block ใหม่ — กรอกข้อมูลซ้ำสำหรับชนิดถัดไป

  4. กดบันทึก

    กดปุ่ม ✅ บันทึกรับเนื้อสด — ระบบจะขึ้น popup แจ้ง "✅ บันทึกสำเร็จ X ถุง / Y.YYY กก."

✅
บันทึกสำเร็จ = ถุงเนื้อจะขึ้นใน สต๊อกปัจจุบัน ทันที สถานะ "✅ In Stock" คลัง A
⚠️
ถ้าขึ้น popup "❌ บันทึกไม่สำเร็จ" ให้ถ่ายรูปหน้าจอแล้วแจ้งไทน์ — ห้ามกดซ้ำ
3 หม้อตุ๋น — เปิด & ปิดหม้อ
📍
ใช้เมื่อ: เริ่มต้มเนื้อ (เปิดหม้อ) และ เสร็จแล้ว (ปิดหม้อ) — ต้องทำทั้งสองขั้นตอน
🛒 รับเนื้อสด 🔥 หม้อตุ๋น ♻️ แปรรูป 📊 สต๊อกปัจจุบัน
เปิดหม้อ — เริ่มต้มเนื้อ
  1. กดปุ่ม 🔥 เปิดเนื้อใหม่

    Kanban board จะโหลดขึ้นมา กดปุ่มสีแดงด้านบน

  2. กรอกใน popup ที่เปิดขึ้น
    กะระบบเลือกให้อัตโนมัติตามเวลา — เช็คว่าถูกต้องก่อน (🌅 เช้า / ☀️ บ่าย / 🌙 เย็น)
    รายการเนื้อเลือกชนิดเนื้อที่จะต้ม — จะเห็นจำนวนถุงที่มีอยู่
    เลือกถุงแตะถุงที่ต้องการ (chip จะเป็นสีน้ำเงินเมื่อเลือก) — เลือกได้หลายถุง
  3. กด 🔥 ส่งเข้าหม้อ

    ระบบจะถามยืนยัน — กด OK — ถุงที่เลือกจะเปลี่ยนสถานะเป็น "❌ Out" และการ์ดหม้อจะขึ้นใน column "🔥 กำลังต้ม"

ปิดหม้อ — เมื่อต้มเสร็จ
  1. กดที่การ์ดหม้อในคอลัมน์ "🔥 กำลังต้ม"

    กด card ของหม้อที่ต้มเสร็จแล้ว — popup "ปิดหม้อ" จะเปิดขึ้น

  2. กรอกผลผลิตที่ได้
    SKU outputเลือกประเภทเนื้อตุ๋นที่ได้ (dropdown MT-)
    จำนวนถุงใส่จำนวนถุงที่ตัดแบ่งได้
    น้ำหนักรายถุงกรอกน้ำหนัก (กรัม) ทีละถุง
    เศษเนื้อ (byproduct)ถ้ามี section สีเหลือง "(บังคับ)" = ต้องกรอก — ใส่น้ำหนักเศษที่ได้
  3. กด "✅ ปิดหม้อ / บันทึก"

    การ์ดจะย้ายไปคอลัมน์ "✅ เสร็จวันนี้" — ถุงเนื้อตุ๋นจะขึ้นในสต๊อก คลัง B/C

📌
วางแผนล่วงหน้า: ถ้าอยากบันทึกว่าจะต้มล็อตไหน แต่ยังไม่เริ่มทันที ให้กด 📝 วางแผนเนื้อ แทน — ถุงจะยัง In Stock จนกว่าจะกดเริ่มจริง
4 แปรรูป / รีแพ็ค
📍
ใช้เมื่อ: ตัดแบ่งเนื้อตุ๋นเป็นถุงขนาดต่างๆ หรือรวม lot / เปลี่ยนขนาดถุง
🛒 รับเนื้อสด 🔥 หม้อตุ๋น ♻️ แปรรูป 📊 สต๊อกปัจจุบัน
ขั้นตอน
  1. กรอกวันที่ และเลือกประเภท
    วันที่ระบบใส่วันนี้ให้ — เปลี่ยนได้
    ประเภทเลือก ครัวกลาง — ตัดแบ่ง/แพ็ค SKU สำหรับการแปรรูปปกติ
  2. เพิ่มถุงต้นทาง (Input)

    กดปุ่ม + เพิ่มถุงต้นทาง — เลือกถุงเนื้อตุ๋นที่จะนำมาแปรรูป — ด้านล่างจะแสดงยอดรวม "รวม input: X.XXX กก."

  3. ระบุผลผลิต (Output)
    SKU outputเลือกชนิดถุงที่แพ็คเสร็จ (เช่น MT-020 พิคาน่า 100g)
    จำนวนถุงใส่จำนวนถุงที่ได้
    น้ำหนักรายถุงกรอกน้ำหนักแต่ละถุง — Yield% จะคำนวณให้อัตโนมัติ
  4. ตรวจสอบ Yield%

    ดูกล่อง Yield ด้านล่าง — สีเขียว = ปกติ / สีส้ม = เตือน / สีแดง = ผิดปกติ ให้แจ้งไทน์

  5. กด ✅ บันทึกแปรรูป

    ถุงต้นทางจะเปลี่ยนสถานะเป็น "♻️ Repacked" — ถุงผลผลิตใหม่จะขึ้นในสต๊อก

⚠️
ถุงที่เลือกเป็น input จะถูกเปลี่ยนสถานะทันทีที่กดบันทึก — ทำซ้ำไม่ได้ ถ้าไม่แน่ใจให้ถามไทน์ก่อน
5 เบิก Non-meat
📍
ใช้เมื่อ: เบิกวัตถุดิบที่ไม่ใช่เนื้อ เช่น ซีอิ๊ว น้ำมัน บะหมี่ เครื่องปรุง บรรจุภัณฑ์ — ต้องบันทึกทุกครั้งที่เบิก
📦 เบิกของ ⚠️ Loss / Waste 📋 Log
ขั้นตอนเบิกของ
  1. เปิดหน้า Stock Dispense → tab 📦 เบิกของ

    stock-dispense.html

  2. ค้นหาวัตถุดิบที่ต้องการเบิก

    พิมพ์ชื่อในช่อง "พิมพ์ชื่อวัตถุดิบ..." — dropdown จะแสดงรายการที่ตรงกัน ใช้ลูกศรขึ้น/ลง + Enter เพื่อเลือก แล้วกด + เพิ่ม

  3. ใส่จำนวนที่เบิก

    แถวที่เพิ่มมาจะแสดง "มี X unit → เหลือ Y" — กรอกจำนวนในช่อง qty — ถ้าตัวเลขเป็นสีส้มหรือขึ้น ⚠️ แปลว่าเบิกเกินสต๊อก

  4. เพิ่มรายการอื่นถ้ามี

    ค้นหาและกด + เพิ่ม ซ้ำได้เรื่อยๆ จนครบทุกรายการที่เบิก

  5. ตรวจสอบวันที่ และกด submit
    วันที่เบิกระบบใส่วันนี้ให้ — เปลี่ยนได้ถ้าบันทึกย้อนหลัง
    หมายเหตุเขียนได้ เช่น "เบิกไปใช้พรุ่งนี้" (ไม่บังคับ)

    กด 📦 ยืนยันเบิกของทั้งหมด — ระบบจะแจ้ง "✅ เบิกแล้ว X รายการ"

บันทึก Loss / Waste — ของเสีย/สูญหาย
📍
ใช้เมื่อ: ของเน่าเสีย / ตกแตก / นับไม่ตรง — ต้องบันทึกทุกครั้งเพื่อ track สต๊อก
  1. กด tab ⚠️ Loss / Waste

    หน้าจะเปลี่ยนเป็น background สีส้ม

  2. ค้นหาและเพิ่มรายการที่เสีย

    เหมือน tab เบิกของ — พิมพ์ชื่อ → กด + เพิ่ม

  3. กรอกจำนวน และเลือกสาเหตุ
    จำนวนใส่ตัวเลขที่เสีย/สูญหาย
    สาเหตุเลือกจาก dropdown: หมดอายุ / เสียหาย / หกหาย / นับไม่ตรง / อื่นๆ
  4. กด ⚠️ บันทึก Loss/Waste ทั้งหมด

    ระบบจะแจ้ง "✅ บันทึก Loss แล้ว X รายการ"

6 ส่งออก (ใบนำส่ง)
📍
ใช้เมื่อ: ส่งเนื้อ + ของ non-meat ออกจากครัวกลาง — ระบบออกใบนำส่งให้ทันที พร้อมปริ้น
📝 เตรียมใบ ⏳ รอส่ง 📋 ประวัติ
🆕
อัปเดต 18/04/2026: เพิ่มแท็บ ⏳ รอส่ง สำหรับจัดการ Draft — บันทึกใบก่อน แก้ไขได้ ค่อยส่งจริงทีหลัง + แท็บ 📋 ประวัติ มีปุ่ม 🖨 พิมพ์ย้อนหลังได้ทุกบิล
📝 แท็บเตรียมใบ — ขั้นตอนกรอก
  1. เปิดหน้า Hub Delivery

    hub-delivery.html

  2. กรอก "ข้อมูลใบนำส่ง"
    วันที่ส่งระบบใส่วันนี้ให้อัตโนมัติ
    ปลายทางเลือก: ครัวกลาง (NT) / หน้าร้าน (FS)
    เลขที่ใบนำส่งระบบเสนอ อัตโนมัติ เช่น NT20260418-1 — ไม่ต้องพิมพ์เอง
  3. Block 1 — เนื้อตุ๋น: เลือก SKU และถุง

    1) เลือก MT item → 2) กด "📦 เลือกถุง" → 3) แตะเลือกถุงใน popup → 4) กด "✅ ยืนยัน"

    หลาย SKU กด + เพิ่มรายการเนื้อ

  4. Block 2 — Non-meat: เลือกสินค้าและจำนวน

    ค้นหา SP item → กรอก qty และหน่วย — รองรับทศนิยม (เช่น ผักบุ้ง 5.07 กก.)

    ของที่ไม่มีใน stock: เลือก MISC-001 รายการพิเศษ → พิมพ์ชื่อในช่อง "ชื่อรายการ (บังคับ)"

  5. เลือกปลายทาง: Draft หรือส่งจริง

    📋 Preview ดูความถูกต้อง — พิมพ์ตัวอย่างได้ด้วยปุ่ม 🖨 พิมพ์

    มี 2 ปุ่มท้ายฟอร์ม:

    • 💾 บันทึก Draft = เก็บไว้ก่อน ยังไม่ตัดสต๊อก ไปแท็บ ⏳ รอส่ง
    • 🚛 บันทึกส่งออก = ส่งจริง ตัดสต๊อกทันที
⏳ แท็บรอส่ง — จัดการ Draft
  1. ดู Draft ที่บันทึกไว้

    Card ส้มแต่ละใบแสดง: เลขบิล → ปลายทาง · วันที่บันทึก · จำนวนเนื้อ/non-meat

  2. คลิก ▼ เพื่อ expand — แก้ไข inline ได้
    • ✏️ แก้จำนวน non-meat
    • 🗑️ ลบรายการออก
    • + เพิ่ม Non-meat ใหม่

    🥩 ถุงเนื้อ (พื้นเหลือง) = read-only — ต้องแก้จากฟอร์ม

  3. ปุ่มบน card
    • 🚛 ส่งออก = โหลด draft เข้าฟอร์ม → ตรวจ Preview → กด 🚛 บันทึกส่งออก → เสร็จแล้ว ลบ draft + เด้งไปแท็บประวัติอัตโนมัติ
    • 🗑 = ลบ draft ทิ้ง (ไม่ส่ง)
📋 แท็บประวัติ — ดูและพิมพ์ย้อนหลัง
  1. เห็นทุกบิลที่ส่งออกแล้ว

    เรียงจากใหม่สุด — header แสดง: บิล · ปลายทาง · วันที่ · 🥩 ถุง/กก. · 🧂 รายการ

  2. คลิก ▼ = ดูรายการในบิล

    🥩 ถุงเนื้อ (พื้นเหลือง) + 🧂 non-meat

  3. ปุ่ม 🖨 = พิมพ์ใบนำส่งย้อนหลัง

    เปิดหน้าต่างใหม่ + auto print dialog — ใช้ reprint บิลเก่าได้ทุกเมื่อ

💡
Flow มาตรฐาน: 📝 เตรียมใบ → 💾 Draft → ⏳ รอส่ง (review) → 🚛 ส่งออก → 📋 ประวัติ (auto)

8 ถ้าติดปัญหา
  1. ลองรีเฟรชหน้าก่อน

    กด F5 บน PC หรือดึงหน้าจอลงบน มือถือ — แก้ได้ ~80% ของปัญหา

  2. ถ่ายภาพหน้าจอที่มีปัญหา
